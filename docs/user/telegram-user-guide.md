# Руководство Пользователя: Telegram

## Что Умеет Бот

Через Telegram-бота можно:

- создать новый чек
- присоединиться к чеку по invite link
- добавить гостя без Telegram-аккаунта в рамках чека
- добавить, обновить и удалить свой расход
- посмотреть список расходов
- запросить итоговый settlement

Все пользовательские команды в MVP работают только в RUB и только с равным делением расхода между выбранными участниками.

## Создание Чека

Создатель отправляет команду:

```text
/new_check Поездка в Казань
```

Бот отвечает:

- подтверждением создания чека
- deep link вида `https://t.me/<bot>?start=join_<invite_token>`

Эту ссылку нужно отправить тем, кто должен присоединиться к чеку.

## Присоединение К Чеку

Участник открывает ссылку и запускает бота.

Технически join выполняется через команду:

```text
/start join_<invite_token>
```

Важно:

- у зарегистрированного участника должен быть настроен Telegram `username`
- без `username` бот вернёт понятную ошибку и не добавит пользователя в чек

## Добавление Гостя

Любой уже присоединившийся зарегистрированный участник может добавить гостя:

```text
/add_guest <invite_token> Маша
```

Ограничения:

- имя гостя должно быть уникальным внутри чека
- guest существует только внутри одного конкретного чека

## Добавление Расхода

Структурированная команда:

```text
/add_expense <invite_token> <amount_minor> <участник1,участник2> | <комментарий>
```

Пример:

```text
/add_expense invite123 1800 me,Маша | Ужин
```

Правила:

- `amount_minor` задаётся в копейках
- `me` означает автора команды
- в split должны быть только участники текущего чека
- расход всегда создаётся от имени автора команды

После успешного создания бот возвращает `expenseId`.

## Просмотр, Изменение И Удаление Расходов

Список расходов:

```text
/list_expenses <invite_token>
```

Изменение своего расхода:

```text
/update_expense <expense_id> <amount_minor> <участник1,участник2> | <комментарий>
```

Удаление своего расхода:

```text
/delete_expense <expense_id>
```

Ограничение MVP:

- менять и удалять можно только собственные расходы

## Редактирование Исходного Сообщения

Если расход был создан командой `/add_expense`, бот использует `telegram_chat_id + telegram_message_id` для синхронизации.

Что происходит дальше:

- если сообщение отредактировано в валидную команду `/add_expense`, данные расхода синхронизируются
- если сообщение стало невалидным, расход уходит в `REQUIRES_CLARIFICATION`
- после повторного корректного редактирования такой расход снова становится `VALID`

Это важно, потому что settlement учитывает только валидные расходы.

## Итоговый Расчёт

Команда:

```text
/settle <invite_token>
```

Ответ содержит:

- текущие балансы участников
- минимальный план переводов

Settlement строится только по расходам со статусом `VALID`.

## Практические Ограничения MVP

- одна валюта: `RUB`
- один расход, один плательщик
- деление расхода только поровну
- внешний публичный API для пользователей отсутствует
- основной интерфейс MVP только Telegram
